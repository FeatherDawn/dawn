线程优先级的特性：继承性、规则性、随机性
【继承性】 线程A启动线程B，线程A和线程B具有同样优先级
【规则性】 CPU尽量将执行的资源让给高优先级的线程用，但不一定是优先级比较大的线程先执行完
【随机性】 即使线程设有优先级，也不能保证线程执行的先后，因为具有随机性

java中分为两种线程
    1、用户线程 - 执行用户级的任务
    2、守护线程 - 后台线程，一般用于执行后台任务

如何区分某一线程是何种线程
    isDaemon()方法来区分: 如果返回false,则说明该线程是用户线程;否则就是守护线程

java虚拟机在“用户线程”都结束后会退出

守护线程基本概念
    1、守护线程是指在程序运行的时候在后台提供一种通用服务的线程
    2、守护线程并不属于程序中不可或缺的部分。入垃圾回收线程。
    3、当所有的用户线程结束时，程序也就终止了，同时会杀死进程中的所有守护线程

用户线程VS守护线程
    如果用户线程已经全部退出运行了，虚拟机也就退出了
    因为没了被守护者，守护线程也就没有工作可做了，也就没有继续运行程序的必要了

守护线程的使用
    1、守护线程并非虚拟机内部特有的
    2、Thread.setDaemon()方法可以设置守护线程
    3、如果想设置线程有守护线程，必须在线程运行前设置，否则会抛IllegalThreadStateException
    4、守护线程创建的子线程也是守护线程
